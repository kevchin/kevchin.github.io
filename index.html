Developing with R and Hadoop

As I've gotten more proficient with R and Hadoop, ironically, having a consistent and reproducible development environment is becoming more of a challenge. With the tool chain always improving and evolving, I always find some other library or resource management tool that promises to make my life easier. Sometimes it's even true ;). In some sense, the ever-changing libraries and technologies suggest a healthy toolchain with regular improvements to libraries and architecure. But as a developer, I want consistency in my environment while I'm coding from one system to another; occasionally I feel I've spent more time configuring systems than developing systems. With the newest generation of tools for managing Virtual Machines (VM's), we now have a relatively easy way to easily create portable development environments.

In this Git Repository , I leverage Vagrant, R, hadoop, and rhadoop to quickly build a consistent R development environment. It a nice way of having something that works and is easily provisioned. Great to run a couple of command, get a coffee and have a ready to use VM.

Hadoop, the Elephant in the Room

With most large production Hadoop clusters, you likely have support resources to help manage the system but if you want to mimic this environment for development on another system (e.g. home), you're own your own. I've used multiple Hadoop starter VM's from vendors but they don't allow me to easily customize a portable development environment. As the development environment extends into the infrastructure via Hadoop, we unfortunately get more involved in the minutae of Hadoop management. Here is where Vagrant comes to the rescue. We use Vagrant to create a VM that has all the tools that we need to do what we need -- in this case R accessing Hadoop and HDFS.

Depending on your Big Data needs and production environment, you may be required to use map reduce streaming. For the most part, I'll leave the Hadoop cluster management to others as long as I can efficiently access the data I need to draw insights.

Azure Machine Learning with R

Azure ML supports R and easily enables me to scale out my experiments. While it is not meant to replace R-Studio, it is another stable R environment that you can leverage. It has 100's of the most common R libraries already installed and ready to use. If you design your R code correctly, you can utilize a "scale-out" model for larger experiments. The basic visualizations are useful but for production work I still prefer to use ggplot.

Python scikit-learn

Python libraries are increasingly powerful and with scikit-learn you can use a number of ML algorithms that are accompanied with well documented examples. If you're familiar with the datasets and data wrangling, you can use those to get a good jump start on your experiments. I prefer simpler examples but rely on python and scikit-learn when I need to manipulate larger streams of data.
